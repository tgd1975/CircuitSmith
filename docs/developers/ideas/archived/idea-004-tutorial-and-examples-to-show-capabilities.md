## Archive Reason

2026-05-13 — Elevated to EPIC-012 (Tutorial and Example Gallery).

A guided tutorial plus a curated gallery of example circuits that
demonstrate, end-to-end, what CircuitSmith can produce today — from
the smallest meaningful `circuit.yml` all the way up to a multi-page
schematic with ERC clean and a BOM that PartsLedger can ingest.

## Motivation

The repo today reads as a set of subsystems and a task plan. A new
contributor (or future-me reopening this in six months) has no fast
path to *seeing* what it does. The dossier in `idea-001.*` describes
the vision; the closed EPIC-001..EPIC-002 tasks describe the
implementation; there is nothing in between that says *"here is a
working input, here is the rendered output, here is what each part of
the file controls."*

This idea is to fill that gap: a `docs/users/tutorial/` walkthrough
plus a `docs/users/examples/` gallery, both rendered to real artefacts
checked into the repo so they double as smoke-test fixtures.

## Rough approach

### Tutorial (linear, ~30 min read)

1. **Minimal circuit** — one resistor, one LED, one source. Author
   `circuit.yml` by hand. Run the skill. See the rendered SVG.
2. **Add a second branch** — introduce a node fan-out, see how the
   netgraph + router handle it.
3. **Introduce a sub-block** — group components into a reusable
   block, show the layout kernel's slot assignment.
4. **Fix an ERC error on purpose** — show what a violation looks like
   in `meta.yml`, fix it, re-run.
5. **Export the BOM** — round-trip into PartsLedger
   (`$CS_PARTSLEDGER_PATH`).
6. **Iterate on layout** — tweak hints, regenerate, diff the SVG.

### Example gallery

A `docs/users/examples/<name>/` directory per example, each containing
`circuit.yml`, the generated `layout.yml` and `meta.yml`, the rendered
SVG, and a short `README.md` describing what makes the example
interesting. Suggested seeds:

- **Voltage divider** — minimal, two-resistor, one-page.
- **Common-emitter amplifier** — one active device, biasing network,
  AC-coupled in/out. Stresses the layout kernel's handling of analog
  signal flow.
- **555 monostable** — single IC with timing components. Stresses
  pin assignment and component grouping.
- **Op-amp non-inverting buffer** — feedback loop, dual-rail supply.
- **Multi-page split** — large enough that the renderer must page-break.

### Mechanics

- Each example is generated by running the skill against its
  `circuit.yml`; CI re-runs them and diffs the outputs, so the gallery
  is also a regression suite.
- The tutorial prose lives in `docs/users/tutorial/` and *embeds*
  excerpts from the examples by relative reference — no copy-pasted
  YAML drifting out of sync.

## Open questions

- Where do user-facing docs live? Today the repo only has
  `docs/developers/`. Do we add `docs/users/`, or fold this under
  `docs/` with a `README` that splits the audiences?
- README on the repo root currently points only at developer docs;
  it should learn to point at the tutorial too.
- Should the tutorial be a Jupyter-style notebook (executable) or
  plain markdown? Notebook gives us live re-runs; markdown is friendlier
  to GitHub rendering.
- Coordination with [[idea-003-detailed-test-plan-for-every-part-of-circuitsmith]] —
  the example gallery should be referenced from the test plan as one
  of the regression-test layers.
- Relationship to PartsLedger: one of the examples should round-trip
  a BOM into PartsLedger to prove the integration works, but that
  pulls a second repo into the docs build. Document the manual path
  first; automate later.
